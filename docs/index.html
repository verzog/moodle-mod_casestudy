Case Study Plugin v1.0

The Case Study activity gives teachers a flexible, field-driven submission form that students fill in and submit for grading. Markers review submissions through a satisfactory / resubmission workflow, and course completion can be tied to the number of accepted submissions.

How it works

Teacher sets up                Student submits              Teacher grades
──────────────────             ──────────────────           ──────────────────────
 • Add activity         ──►    • Fills in fields     ──►    • Opens submission
 • Add fields                  • Saves as draft              • Writes feedback
 • Configure settings          • Clicks Submit               • Marks Satisfactory
                                                               ──── or ────
                                                              • Requests Resubmission
                               ◄── Student notified ────────────────────────────
                                                ↕
                               Student resubmits (if requested)

Key concepts

Submission statuses

StatusMeaning
DraftSaved but not yet submitted. Only the student can see it.
SubmittedStudent finalised submission. Awaiting grader review.
In ReviewA grader has opened the submission and is marking it.
Awaiting ResubmissionGrader has requested the student revise and resubmit.
ResubmittedStudent has submitted a revised version.
SatisfactoryGrader accepted the submission. ✓
UnsatisfactoryGrader did not accept the submission.
